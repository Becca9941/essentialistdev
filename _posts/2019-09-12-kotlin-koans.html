Function declaration documentation

  • Funtions are declared using the 'fun' keyword.
  • You can call functions using the traditional approach: double(2)
  • Member functions (that belong to a class) can be called using the dot notation: Stream().read()
  • Function parameters must be explicitly typed 'name: type' fun powerOf(number: Int, exponent: Int) {}
  • Default arguments: default values if argument is not provided when one is expected 'off: Int = 0'
  • Overriding methods have the same default values as base method. So don't include default parameters in the override method
  • If you have a function with two arguments, and the first argument has a default value but the second argument does not, the default value can only be used if you call the function using named arguments: say(greeting = "Hi") where the default bye argument was "Goodbye".
  • I struggled to understand the lambda section of the documentation, so had a chat with CTO who said that lambdas are anonymous functions, and they can be passed as arguments to functions. Map, reduce and filter are examples of lambdas. In map for example you might have an array of numbers and say arr.map(square) - this isn't the proper syntax. Then this will return an array of squared numbers from the previous array. You can also write your own lambdas and pass them as parameters in your own functions.
